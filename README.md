# StudyBuddy

StudyBuddy — это веб-приложение «Тамагочи для студентов», 
которое помогает отслеживать расписание, посещаемость и дедлайны.

## Стек технологий

- Frontend: React + Vite
- Backend: Node.js + Express
- DevOps: Docker + Docker Compose
- Optional: Telegram Bot на Python

## Запуск проекта

### Локально через Docker

При первом запуске / изменениях файлов
```bash
    docker-compose up --build
```

Простой запуск
```bash
    docker-compose up 
```

Сайт доступен по адресу: http://localhost:5173/

### Вручную (NPM Scripts)

Backend:
- npm run dev — запуск с nodemon
- npm start — обычный запуск

Frontend:
- npm run dev — запуск Vite

##  Архитектура приложения

StudyBuddy построен по классической клиент-серверной архитектуре с использованием REST API.

### 1. Клиентский уровень (Frontend)

Технологии:
- React
- Vite

Основные требования:
- Отображение пользовательского интерфейса
- Обработка действий пользователя
- Отправка HTTP-запросов к серверу
- Получение и отображение JSON-ответов

Frontend отвечает только за UI и взаимодействие с API.

### 2. Серверный уровень (Backend)

Технологии:
- Node.js
- Express

Основные требования:
- Обработка HTTP-запросов
- Проверка и валидация данных
- Реализация различной логики
- Формирование JSON-ответов
- Управление состоянием приложения
- Работа с базой данных

Backend является центральным уровнем логики приложения.

### 3. Модель взаимодействия

Последовательность работы:

1. Пользователь выполняет действие в браузере / telegram боте
2. Frontend / telegram bot отправляет HTTP-запрос на backend
3. Backend обрабатывает запрос
4. Backend возвращает JSON-ответ
5. Frontend / telegram bot обновляет интерфейс
